# Bugs And Risks

## 已知问题

- 商品编辑页图片分批上传时，旧实现每上传成功一张就直接基于页面当前 `form` 写回，缺少批次级合并入口；在连续选择多批图片时表现为后批图片覆盖或主图未回填。

## 风险记录

- 需要在微信开发者工具里再手动验证真实 `wx.cloud.uploadFile` 行为；Node 测试只能覆盖模型合并逻辑和页面脚本语法。

## 失败尝试

- 上一版仅使用 `appendProductImage` 逐张追加，未彻底解决用户反馈的分批上传覆盖问题。

## 待验证项

- 在开发者工具中编辑同一商品：先上传 1 张，再上传 1 张或多张，保存后产品列表和详情页应显示主图与全部花色图。

## 需要用户确认的事项

- 无。

## 结论

- 根因按当前代码判断为上传阶段缺少批次级图片合并与唯一批次路径；已改为上传完成后一次性将新批次合并到当前表单图片列表。
