# Bugs And Risks

## 已知问题

- 暂无新增已知问题。

## 风险记录

- 新增 `settings` 云函数后，必须在微信开发者工具中上传并部署，否则前端会回退到本地默认门店和分类配置。
- 需要在微信开发者工具里手动验证门店二维码、门店实拍图上传以及云端分类回显。

## 失败尝试

- 无。

## 待验证项

- 部署 `settings` 云函数后，进入商铺管理分别保存门店信息和分类，确认首页、产品页、门店页和商品表单回显。

## 需要用户确认的事项

- 无。

## 结论

- 门店和分类管理使用独立 `settings` 云函数，前端保留本地 fallback，降低未部署或云端异常时的影响。
