# Bugs And Risks

## 已知问题

- 商品编辑页此前在 `onShow` 中重新执行权限校验和商品加载；从相册/图片选择器返回时可能触发页面显示，导致未保存的上传图片被数据库里的旧商品数据覆盖。

## 风险记录

- 新增 `settings` 云函数后，必须在微信开发者工具中上传并部署，否则前端会回退到本地默认门店和分类配置。
- 商品编辑页上传图片需要在微信开发者工具中再做真实 `wx.chooseImage` + `wx.cloud.uploadFile` 手动验证。

## 失败尝试

- 仅修复图片数组合并不能覆盖生命周期重载导致的表单覆盖问题。

## 待验证项

- 在开发者工具中进入已有商品编辑页，上传图片后不保存先停留在页面，确认图片仍保留；保存后确认产品列表和详情页显示。

## 需要用户确认的事项

- 无。

## 结论

- 商品编辑页初始化应放在 `onLoad`，不能在 `onShow` 中反复重新加载商品详情。
