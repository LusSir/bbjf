# Bugs And Risks

## 已知问题

- 暂无本次实现中已确认的运行时 bug。

## 风险记录

- `orders` 云函数新增后必须在微信开发者工具中上传并部署，否则小程序端调用会报函数不存在。
- 云数据库集合由云函数尝试自动创建；如果云环境权限或集合规则限制创建，需要在控制台手动创建 `orders` 集合。
- 订单手机号仅做基础 11 位手机号格式校验，不包含短信验证。
- 商品 SKU 库存是文案展示，不做真实库存扣减。
- 当前验证主要是模型测试、语法检查和 JSON 校验，仍需在微信开发者工具里走完整手动流程。

## 失败尝试

- 暂无本次实现中的失败尝试。

## 待验证项

- 游客加购物车后，提交订单时登录并填写联系方式。
- 用户能在我的订单看到订单状态变化。
- 管理员能按状态筛选订单并更新为已确认、备货中、已完成或已关闭。
- 老商品没有 SKU 时仍能加入购物车。
- SKU 停用后不能继续提交新订单，历史订单仍保留快照。

## 需要用户确认的事项

- 暂无。

## 结论

- 本次功能静态验证已通过，真实小程序云开发链路需要在微信开发者工具中继续验收。
