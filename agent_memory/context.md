# Project Context

## 项目概览

- 项目名称：贝贝家纺微信小程序。
- 项目目标：实体店成品家纺商品展示、门店联系、购物车预约订单、用户订单查询和管理员后台维护。
- 主要技术栈：微信小程序原生页面、微信云开发、Node.js 云函数。
- 运行入口：微信开发者工具打开项目根目录。
- 测试入口：`node scripts/test-*.js` 与相关 `node --check`。

## 关键路径

- 源码：`miniprogram/`、`cloudfunctions/`。
- 测试：`scripts/`。
- 配置：`project.config.json`、`miniprogram/app.json`。
- 文档：`docs/`、`agent_memory/`。

## 本地约定

- 代码风格遵循仓库现有风格。
- 优先使用仓库已有 scripts 和原生小程序能力。
- 改动后运行与范围匹配的最小必要验证。

## 已确认事实

- 商品管理页支持商品多图，`form.images` 保存花色图列表，`form.image` 保存主图。
- 商品图片上传使用 `wx.cloud.uploadFile`。
- 门店信息和分类配置通过 `settings` 云函数读写云数据库。
- 登录通过 `login` 云函数写入/更新 `users` 集合，管理员通过 `role === "admin"` 识别。
- 订单第一版是预约确认订单，不做在线支付、退款、物流和真实库存扣减。
- 购物车使用本地存储 `bbjf_cart`，提交订单时由 `orders` 云函数重新校验商品和 SKU。

## 当前假设

- 第一版以手动录入商品为主，不继续做 Excel 批量导入。
- 分类不做物理删除，使用启用/停用保护已有商品引用。
- 管理员提醒先通过后台订单列表查看，不做订阅消息。

## 注意事项

- 不硬编码密钥。
- 不提交 `.env`。
- 不在日志中泄露敏感信息。
- 修改公共 API / 数据结构 / 数据库 schema / 删除文件前先确认。
