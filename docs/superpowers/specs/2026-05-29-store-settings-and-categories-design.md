# Store Settings And Categories Design

## Goal

完善商铺后台，让管理员可以在小程序里维护门店信息和商品分类；前台页面优先读取云端配置，云端不可用时继续使用本地默认配置。

## Scope

- 新增 `settings` 云函数，负责门店信息与分类的读取、保存。
- 新增 `store_settings` 集合，保存单条门店配置。
- 新增 `categories` 集合，保存商品分类配置。
- 新增门店信息管理页和分类管理页。
- 首页、产品页、门店页、商品管理页、商品编辑页改为优先读取云端分类/门店信息。

## Decisions

- 分类内部 id 由云函数生成，管理员不需要填写。
- 分类第一版不做物理删除，使用启用/停用，避免已有商品引用的分类丢失。
- 门店信息支持维护名称、标语、电话、地址、营业时间、经纬度、微信二维码和门店实拍图。
- 保留本地 `config/store.js` 与 `data/categories.js` 作为 fallback，降低云函数未部署或云数据库未初始化时的影响。

## Validation

- 使用 Node 脚本覆盖门店/分类模型规则。
- 使用 `node --check` 检查新增和修改的 JS 文件。
- 运行已有商品、权限、云错误相关测试，确认没有回归。
