# 贝贝家纺微信小程序第一版设计

## 背景

贝贝家纺是实体家纺门店。第一版微信小程序定位为获客导购型产品展示工具，重点解决用户看产品、联系门店、电话咨询、导航到店的问题。小程序暂未注册，因此第一版先采用可本地开发和预览的轻量方案，避免把后台、支付、会员等能力提前做重。

## 目标

- 展示门店主营成品家纺产品。
- 让用户能按品类浏览商品。
- 用热卖、特价、新品辅助导购。
- 让用户能快速微信咨询、电话联系、导航到店。
- 保持第一版简单，后续再迭代后台、客服、活动和线上交易。

## 非目标

- 不做在线支付。
- 不做购物车。
- 不做会员登录。
- 不做复杂 SKU。
- 不做定制预约。
- 不做店员后台。
- 不做库存系统。

## 技术路线

第一版使用原生微信小程序开发。商品和分类数据先内置在代码文件中，图片放在本地 `assets/` 目录。门店信息集中放在配置文件中。后续如果商品数量变多或需要店员维护，再迁移到微信云开发数据库或独立后台。

推荐目录：

```text
miniprogram/

```

## 页面结构

### 首页

首页采用品类优先结构。首屏展示贝贝家纺品牌、实体店可信信息和快捷动作，核心区域展示产品分类入口，辅助区域展示热卖、特价、新品。

快捷动作包括：

- 微信咨询
- 电话联系
- 导航到店

### 产品分类页

产品页按品类浏览商品。第一版建议品类包括：

- 床品四件套
- 被芯
- 枕芯
- 毯子
- 凉席
- 婚庆套件
- 特价专区

如果某个分类暂无商品，展示空状态文案：`该分类商品正在整理中，可先微信咨询或到店看看`。

### 商品详情页

商品详情页展示商品图片、名称、卖点、规格、适用场景和价格信息。底部固定展示咨询、电话、导航按钮。

价格策略采用部分展示：

- 热卖、特价、新品可以展示明确价格、起价或区间。
- 普通商品默认展示 `到店咨询价` 或 `微信咨询价`。
- 页面提示 `花色、尺寸、库存以门店实物为准`。

### 门店页

门店页展示地址、营业时间、联系电话、微信咨询方式、门店实拍或门头图，并提供地图导航入口。

### 联系页

第一版不做独立账号系统。联系能力可合并在门店页，也可以从首页和商品详情页按钮进入。内容包括微信咨询提示、电话、地址、营业时间和常见问题。

## 导航结构

底部 Tab 使用 3 个入口：

- 首页
- 产品
- 门店

联系能力通过页面按钮和门店页承载，避免第一版导航过重。

## 关键交互

### 微信咨询

小程序注册和客服能力配置完成前，先采用复制微信号加提示说明的方式。后续可以替换为微信客服入口。

### 电话联系

使用微信小程序原生拨打电话能力，点击后调起拨号。

### 导航到店

使用微信小程序地图能力打开导航。正式发布前需要补齐门店经纬度。

### 商品图片

商品列表展示封面图，详情页展示多图轮播。开发阶段可使用占位图，正式上线建议替换成真实商品图和门店实拍图。

### 分享

第一版支持分享首页和商品详情。分享文案突出实体店和到店看实物，例如：`贝贝家纺成品床品，到店可看实物`。

## 验证计划

- 在微信开发者工具中打开项目。
- 验证首页、产品页、商品详情页、门店页可访问。
- 验证分类切换和商品详情跳转。
- 验证热卖、特价、新品展示。
- 验证空分类状态。
- 验证复制微信号。
- 验证电话拨打入口。
- 验证地图导航入口。
- 验证分享标题和路径。
- 真机扫码预览核心流程。

## 注册和上线流程

1. 到微信公众平台注册小程序账号。
2. 按营业执照选择主体类型，实体店通常为个体工商户或企业。
3. 完成主体信息、小程序名称、头像、简介、服务类目。
4. 安装微信开发者工具。
5. 获取 AppID 后导入项目；未注册前可先使用测试号或无 AppID 开发预览。
6. 配置隐私说明、服务类目和必要权限。
7. 上传代码版本。
8. 在微信公众平台提交审核。
9. 审核通过后发布。

## 正式发布前所需资料

- 门店完整地址。
- 联系电话。
- 微信号或二维码。
- 营业时间。
- 主营品类。
- 第一批商品名称、图片、价格策略和卖点。
- 门头或门店实拍图。
- 小程序最终名称。
